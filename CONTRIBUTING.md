## Contributor Tiers & Commercial Rights

We recognize and reward contributors with commercial use rights based on their level of contribution:

| Tier | Requirements | Rights Granted |
|------|--------------|----------------|
| **Contributor** | 1+ merged PR | Attribution in Contributors list |
| **Core Contributor** | 3+ significant PRs *or* 1 new chapter | Personal commercial use rights |
| **Co-Author** | Major contributions (multiple chapters, extensive rewrites) | Full commercial rights + co-author credit |

### What Counts as "Significant"

- New sections with substantial content (500+ words)
- Major improvements to existing chapters
- New examples, diagrams, or code samples
- Translations of chapters
- Bug fixes and corrections count toward the total but are weighted less

### Commercial Rights Details

**Core Contributors receive:**
- Right to use this guide in paid training/workshops they conduct
- Right to include excerpts in commercial publications (with attribution)
- Right to create derivative works for commercial purposes

**These rights are:**
- Personal and non-transferable
- Granted in writing via GitHub (your merged PRs serve as record)
- Subject to maintaining proper attribution

## How to Contribute

### 1. Fork the Repository

```bash
# Fork via GitHub UI, then clone your fork
git clone https://github.com/YOUR-USERNAME/ai-and-claude-code-intro.git
cd ai-and-claude-code-intro
```

### 2. Create a Branch

```bash
git checkout -b feature/your-contribution-name
```

### 3. Make Your Changes

Follow the style guidelines below.

### 4. Submit a Pull Request

- Provide a clear description of your changes
- Reference any related issues
- Ensure your PR passes any checks

## Style Guidelines

### Writing Style

- **Audience**: DevOps engineers, SREs, platform engineers
- **Tone**: Professional but approachable, practical over theoretical
- **Examples**: Include real-world DevOps examples wherever possible
- **Code**: All code examples should be tested and working

### Markdown Formatting

```markdown
# Chapter Title (H1 - one per file)

## Major Sections (H2)

### Subsections (H3)

**Bold** for emphasis
`code` for inline code
```bash for code blocks with language specified
```

### File Naming

- Chapters: `XX-descriptive-name.md` (e.g., `11-new-topic.md`)
- Keep names lowercase with hyphens
- Presentations follow same pattern in `presentations/`

### Content Requirements

- Include practical examples
- Add DevOps-specific context where relevant
- Provide "Key Takeaways" at end of chapters
- Include exercises when appropriate

## Review Process

1. **Initial Review**: Maintainer reviews for quality and fit
2. **Feedback**: You may receive requests for changes
3. **Approval**: Once approved, PR is merged
4. **Attribution**: You're added to Contributors list

Typical review time: 1-2 weeks
